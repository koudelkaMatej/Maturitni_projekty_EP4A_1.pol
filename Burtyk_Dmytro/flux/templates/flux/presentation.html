Project Overview

Flux Electronics: Uživatelský manuál

Kompletní průvodce procházením, správou účtu a nakupováním na naší platformě.

1. Účet a zabezpečení

Spravujte své osobní údaje bezpečně a jednoduše.

1
Registrace a bezpečnost: Vytvořte si účet bezpečně. Náš systém obsahuje kontrolu síly hesla, která zajistí ochranu vašich dat.
2
Profil a uložené údaje: Své údaje můžete kdykoli upravit. Uložte si výchozí doručovací adresu, aby se při platbě vyplnila automaticky!
3
Smazání účtu: Máte plnou kontrolu nad svými daty. Svůj účet a všechny s ním spojené informace můžete trvale smazat v nastavení profilu.

2. Vyhledávání produktů

Naše chytrá navigace vám pomůže najít přesně to, co hledáte.

1
Chytré vyhledávání: Víte přesně, co chcete? Napište název produktu do vyhledávacího pole v horní části stránky a najděte jej okamžitě.
2
Dynamický filtr kategorií: Pomocí bočního menu můžete procházet konkrétní oddělení (např. Notebooky, Telefony). Aktivní kategorie zůstane zvýrazněna.
3
Produktové karty: Přejetím myší přes produkt uvidíte 3D efekt. Po kliknutí se zobrazí detailní specifikace.

3. Vytvoření objednávky

Plynulý a bezpečný proces nákupu.

1
Správa košíku: Zkontrolujte své položky, upravte množství nebo odstraňte produkty. Celková cena se automaticky přepočítá.
2
Automatické vyplňování: Při přechodu k pokladně systém automaticky načte vaši uloženou adresu z profilu, aby vám ušetřil čas!
3
Možnosti doručení: Před kliknutím na „Odeslat objednávku“ si vyberte standardní doručení, lokální kurýra nebo bezplatné vyzvednutí na prodejně.

4. Potvrzení objednávky

Buďte informováni o každém kroku.

1
Systémová upozornění: Všímejte si zelených zpráv v horní části obrazovky, které potvrzují úspěšné vytvoření objednávky.
2
E-mailová účtenka: Jakmile je objednávka vytvořena, systém automaticky odešle detailní účtenku přímo na váš ověřený e-mail!
Investiční příležitost

Flux Electronics: Architektura budoucnosti

Moderní, bezpečná a plně škálovatelná e-commerce platforma připravená na masivní růst uživatelské základny a minimalizaci provozních nákladů.

Zabezpečení na úrovni bank

Ochrana proti SQL injection, CSRF útokům a bezpečné šifrování hesel pomocí algoritmu PBKDF2.

Vysoká konverze (UX)

Frikční nákupní proces, dynamické filtrování a blesková navigace zvyšují procento dokončených nákupů.

Škálovatelný Backend

Postaveno na robustním frameworku Django. Připraveno na integraci API, tisíce produktů a vysokou zátěž.

Ochrana uživatelských dat jako priorita

V dnešní době je bezpečnost dat největším rizikem pro e-commerce. Náš systém je od základu navržen tak, aby chránil jak firmu, tak zákazníka.

  • Pokročilá validace hesel: Neukládáme hesla v čistém textu, využíváme jednosměrné kryptografické hašování.
  • Ochrana relací (Session Security): Každý formulář je chráněn unikátním CSRF tokenem.
  • Bezpečná správa databáze: Striktní oddělení uživatelských a administrátorských práv.

Maximalizace dosahu díky UX a Mobile-First

Zákazníci nečekají. Pokud nenajdou produkt do 3 sekund, odcházejí ke konkurenci. Náš front-end je optimalizován pro okamžitou odezvu.

  • Blesková navigace a vyhledávání: Uživatelé mohou okamžitě přepínat mezi kategoriemi pomocí dynamického postranního panelu nebo využít přesné textové vyhledávání.
  • 100% Responzivní design: Díky technologii Bootstrap 5 se e-shop plynule přizpůsobí jakémukoliv zařízení – od 4K monitorů po malé obrazovky smartphonů.

Odstranění bariér v nákupním procesu

Systém aktivně pracuje na tom, aby se minimalizovalo opuštění košíku. Dali jsme zákazníkům plnou kontrolu a flexibilitu.

  • Smart Checkout: Přihlášeným uživatelům systém automaticky předvyplňuje doručovací údaje. Méně psaní znamená více dokončených objednávek.
  • Flexibilní doručení a platby: Zákazník má na výběr z více možností (Standardní doručení, Lokální kurýr, Vyzvednutí na prodejně), přičemž systém dynamicky přepočítává celkovou cenu.
  • Automatizované e-mailové účtenky: Okamžité potvrzení objednávky snižuje zátěž zákaznické podpory o desítky procent.

Nulové náklady na vývoj Back-Office

Vývoj administračního rozhraní na míru často zdvojnásobuje rozpočet projektu. My jsme tento problém elegantně vyřešili pomocí nativních nástrojů.

  • Django Admin Panel: Integrovaný, plně zabezpečený administrační systém pro správu eshopu, který nevyžadoval žádné dodatečné náklady na vývoj.
  • Efektivní správa katalogu: Přidávání nových produktů, kategorií a úprava cen je záležitostí několika kliknutí přímo z prohlížeče.
  • Centralizovaná správa uživatelů: Kompletní přehled nad uživatelskými účty a oprávněními z jednoho místa, chráněný superuživatelským přístupem.
Developer Onboarding

Flux Electronics: Technický přehled

Vítej v týmu! Tento dokument ti pomůže rychle se zorientovat v architektuře projektu, databázovém schématu a pravidlech vývoje.

1. První spuštění projektu

Než začneš programovat, musíš si projekt rozběhnout u sebe na počítači. Zde je postup, jak nastavit lokální vývojové prostředí:

  1. Vytvoření virtuálního prostředí: Odděluje balíčky tohoto projektu od zbytku PC.
  2. Aktivace prostředí: Musíš provést před každou prací na projektu.
  3. Instalace závislostí: Stáhne Django z requirements.txt.
  4. Migrace databáze: Vytvoří prázdné tabulky v lokální databázi.
  5. Spuštění serveru: Zapne web na localhost:8000.
# 1. Vytvoření a aktivace prostředí (Windows)
> python -m venv venv
> .\venv\Scripts\activate
# 2. Instalace knihoven a migrace
> pip install -r requirements.txt
> python manage.py migrate
# 3. Zapnutí serveru
> python manage.py runserver

2. Architektura složek

Abychom udrželi kód čistý, striktně oddělujeme logiku, šablony a statické soubory:

  • Hlavní aplikace (např. flux/): Zde žije veškerá Python logika (views, models, urls).
  • templates/: Zde jsou všechny HTML soubory. Používáme dědičnost z base.html.
  • static/: CSS soubory a statické obrázky rozhraní.
  • media/: Sem se automaticky ukládají fotky produktů nahrané přes Admin panel.
flux_project/
core/ (settings.py, urls.py)
flux/ (modely a views)
media/ (fotky produktů z adminu)
static/ (css a layout obrázky)
templates/ (HTML šablony)
manage.py
db.sqlite3

3. Databázová architektura a vztahy

Projekt využívá relační databázi spravovanou přes Django ORM. Níže uvedený ERD (Entity-Relationship Diagram) znázorňuje propojení jednotlivých tabulek a jejich kardinalitu:

User & Order (Vztah 1:N)

Jeden uživatel může mít více objednávek. Tabulka Order na něj odkazuje přes Cizí klíč (User ID) a uchovává specifické doručovací údaje pro daný nákup.

Order & Product (Vztah M:N)

Propojení řeší asociativní entita Order items. Ta drží cizí klíče obou tabulek, uchovává zakoupené množství (Quantity) a dynamicky počítá celkovou cenu položky.

Category & Product (Vztah 1:N)

Jedna kategorie může obsahovat nekonečně mnoho produktů. Každý model Product má Cizí klíč odkazující na svou domovskou kategorii.

Product & ProductImage (Vztah 1:N)

Pro oddělení logiky může mít jeden produkt více fotek v galerii. Tabulka ProductImage se váže na konkrétní produkt přes jeho ID.

4. Administrační rozhraní

Pro správu obsahu poskytuje Django výkonný Admin panel zaregistrovaný v admin.py.

Jak se dostat do Adminu:
  1. Běž na adresu 127.0.0.1:8000/admin.
  2. Pokud nemáš účet, vytvoř ho:
    python manage.py createsuperuser

5. Zlatá pravidla pro psaní kódu

1. Neupravuj databázi bez migrací!

Pokud přidáš sloupec do models.py, musíš spustit makemigrations a migrate.

2. Vypínej DEBUG v produkci!

V settings.py musí být DEBUG = False před nasazením na reálný server kvůli bezpečnosti.

3. Hardcoding URL je zakázán!

V HTML šablonách nepiš href="/shop". Používej dynamický tag {% url 'název' %}.

6. Mapa funkcí (Pod kapotou)

Tento tahák ti pomůže pochopit pokročilou logiku a najít správný soubor, když budeš upravovat kód.

Kategorie a Vyhledávání

Filtrování produktů na hlavní stránce obchodu.

  • Logika: views.py (zpracování parametru `?q=`)
  • Routy: urls.py
Context Processors

Jak dostáváme data (např. počet položek v košíku) do všech šablon najednou bez psaní kódu do každého view.

  • Soubor: context_processors.py
  • Nastavení: Musí být registrováno v settings.py v sekci TEMPLATES.
Logika košíku (cart.py)

Košík nevyužívá databázi, dokud uživatel nezaplatí. Místo toho využíváme Django Sessions.

  • Třída Cart: V souboru cart.py držíme logiku pro přidávání a počítání cen.
  • Ukládání: Data žijí v request.session.
Logování a Debugging

Jak sledujeme chyby v produkci (např. selhání odeslání e-mailu nebo chyby v platbě).

  • Konfigurace: Slovník LOGGING v settings.py.
  • Zápis: Události se zapisují do souboru (např. debug.log), abychom je mohli zpětně analyzovat.

7. Verzování a Git Flow

Na projektu pracujeme v týmu, proto striktně dodržujeme pravidla pro nahrávání kódu na GitHub.

Soubor .gitignore

Tento soubor říká Gitu, které soubory má zcela ignorovat a nenahrávat na server. Jde o bezpečnost!

  • db.sqlite3: Nikdy nenahráváme produkční databázi!
  • /venv/: Složka s virtuálním prostředím má stovky megabajtů, ignorujeme ji.
  • .env (Tajné klíče): Hesla a Django SECRET_KEY nikdy nesmí jít na GitHub!
# Správný postup pro nahrání kódu (Git Push)
# 1. Zkontroluj, co jsi změnil
> git status
# 2. Přidej všechny bezpečné soubory
> git add .
# 3. Vytvoř balíček s popisnou zprávou
> git commit -m "feat: Přidána logika pro nákupní košík"
# 4. Odešli kód bezpečně na hlavní větev
> git push origin main
