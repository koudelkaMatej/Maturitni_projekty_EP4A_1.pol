Průvodce pro spolupracovníka

Technická dokumentace architektury, kódu a vývojového workflow projektu DRIVE.

Přehled architektury

🖥️ Backend (Node.js + Express 5)

  • Jeden soubor server.js (~1 300 řádků)
  • REST API pro produkty, košík, objednávky, auth
  • Synchronní SQLite přes better-sqlite3
  • PDF faktury přes pdfkit
  • E-maily přes nodemailer
  • Strukturované logování do souborů (src/logger.js)

🌐 Frontend (Vanilla JS)

  • Statické HTML stránky servírované Expressem
  • Žádný framework (React/Vue/Angular)
  • CSS custom properties, žádný Tailwind/Bootstrap
  • Moduly: cart.js, products.js, dev-mode.js
  • Fonts: TelkaTRIAL (Medium + Bold, otf)
Architekturní rozhodnutí: MPA (Multi-Page Application) místo SPA. Každá stránka je samostatný HTML soubor. Navigace = normální HTTP redirect.

Autentizace

Uživatelské session

Cookie auth_token (httpOnly, sameSite: lax, 30 dní). Token je UUID v4, uložen v tabulce sessions s expirací.

getCurrentUser(req) { const token = req.cookies?.auth_token; if (!token) return null; return db.prepare( 'SELECT u.* FROM sessions s JOIN users u...' ).get(token, new Date().toISOString()); }

Admin / Dev mode

Dvě cookies:
dev_mode — nesigned, pro UI viditelnost
dev_token — signed httpOnly, pro API ochranu

// API ochrana if (req.signedCookies.dev_token !== 'authorized') { return res.status(403).json({...}); }
Rate limiting: Login endpoint je chráněn in-memory rate limiterem — max 5 pokusů / 15 minut per IP.

Vzory a konvence

Logger

Vždy používej logger místo console.*. Loguje do souborů s JSON formátem.

const logger = require('./src/logger'); logger.info('checkout', 'Order created', { orderId }); logger.error('auth', 'Login failed', { email });

DB transakce

Pro vícekrokové operace vždy použij db.transaction() — atomičnost a výkon.

const createOrder = db.transaction(() => { // 1. Ověř sklad // 2. Vytvoř objednávku // 3. Odečti sklad return orderId; }); const id = createOrder();

Ceny vždy v haléřích

Databáze ukládá ceny jako integer haléře. Frontend dělí 100.

// Správně: price_cents: 3990 // = 39,90 Kč // Frontend zobrazení: (price_cents / 100).toFixed(0) + ' Kč'

Error handling

API vrací konzistentní JSON chyby. Vždy loguj chyby s kontextem.

} catch (e) { logger.error('ctx', 'msg', { message: e.message }); res.status(500).json({ error: '...' }); }

Košík — dual-mode

Košík funguje ve dvou módech souběžně. Při přihlášení se automaticky synchronizuje localStorage košík se serverovým.

📦 Guest (localStorage)

Nepřihlášený uživatel ukládá košík do localStorage jako JSON pole. Rychlé, bez API volání.

☁️ Přihlášený (server)

Po přihlášení se localStorage košík synchronizuje přes POST /api/user/cart/sync na server. Data přetrvají přes prohlížeče.

Jak přispívat

Větve a workflow

  • Pracuj na feature větvi: git checkout -b feature/popis
  • Commituj malé logické celky s popisným mesagem
  • PR merge do main větve

Přidání nové API route

  1. Najdi správnou sekci v server.js (komentáře // ===== API: XXX =====)
  2. Přidej route s app.get/post/put/delete
  3. Pro admin routes přidej kontrolu req.signedCookies.dev_token
  4. Všechny chyby loguj přes logger.error('context', 'msg', data)
  5. Nezapomeň na vstupní validaci a odpovědi 400/401/403/404/500

Přidání nové HTML stránky

  1. Vytvoř HTML soubor ve správné složce
  2. Přidej <script src="/assets/js/dev-mode.js"></script> před </body>
  3. Pokud má navigaci, použij id="navMenu" na <ul>
