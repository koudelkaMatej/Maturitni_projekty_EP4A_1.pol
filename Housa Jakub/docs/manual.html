Uživatelský manuál

Kompletní průvodce projektem DRIVE. — od instalace po deployment

1. Přehled projektu

DRIVE. je e-shop pro energetické nápoje postavený jako Multi-Page Application (MPA) s vanilla JavaScriptem (bez frameworku). Server běží na Node.js + Express 5, data jsou uložena v SQLite databázi.

E-shop funkce

  • Katalog produktů z databáze
  • Detail s přepínáním příchutí
  • 4-krokový checkout
  • Slevové kódy & PDF faktura
  • Uživatelský účet + předplatné

Uživatelé

  • Registrace / přihlášení
  • Reset hesla e-mailem
  • Historie objednávek
  • Správa předplatného
  • GDPR smazání účtu

Backend

  • Newsletter s kódem DRIVE10
  • Admin rozhraní (dev mód)
  • GDPR cookie consent
  • Responzivní design
  • Vlastní font TelkaTRIAL

6. Systém košíku

Nepřihlášený uživatel

Košík se ukládá do localStorage v prohlížeči. Při načtení stránky se synchronizuje se serverovým košíkem přes session ID (cookie).

Serverový košík (/api/cart) je vázán na session cookie — funguje i bez localStorage jako záloha.

Přihlášený uživatel

Košík se ukládá do databáze (tabulky carts + cart_items). Při přihlášení se automaticky sloučí localStorage košík se serverovým košíkem.

Endpoint POST /api/user/cart/sync provede sloučení a vrátí nový stav košíku.

Předplatné v košíku

Položka s is_subscription: true označuje měsíční předplatné. Po dokončení checkoutu se vytvoří záznam v tabulce user_subscriptions a nastaví se next_billing_date o 30 dní do budoucna.

7. Objednávkový proces

4 kroky checkoutu

  1. Košík — přehled položek, možnost změny množství, zadání slevového kódu
  2. Doručení — jméno, adresa, telefon, e-mail (předvyplněno z profilu pokud přihlášen)
  3. Platba — výběr metody (karta / bankovní převod / dobírka)
  4. Shrnutí — rekapitulace objednávky před odesláním

Po odeslání objednávky

  • Vytvoří se záznam v orders + order_items
  • Sníží se stock u produktů
  • Odešle se potvrzovací e-mail
  • Vygeneruje se PDF faktura
  • Vymaže se košík

Slevové kódy

Ověření přes POST /api/validate-discount. Kód musí být v tabulce discounts s active = 1.

Výchozí kód: DRIVE10 (10% sleva) — seedován automaticky při prvním spuštění.

8. Autentizace a uživatelské účty

Jak funguje přihlášení

  1. Uživatel odešle e-mail + heslo na POST /api/auth/login
  2. Server ověří heslo přes bcryptjs.compare() (hash uložen v DB)
  3. Vytvoří se session token (UUID v4), uloží do tabulky sessions s expirací 30 dní
  4. Token se nastaví jako signed httpOnly cookie session_token

Reset hesla

Uživatel požádá o reset → server vygeneruje token a uloží do password_resets s platností 1 hodinu → odešle e-mail s odkazem → uživatel zadá nové heslo → token se smaže.

GDPR smazání účtu

DELETE /api/user/delete — anonymizuje objednávky (nastaví user_id = NULL), smaže session tokeny, zruší předplatné a odstraní záznam z users.

SMTP konfigurace

Nakonfigurujte proměnné SMTP_* v souboru .env. Pro testování použijte Ethereal Email — zprávy se zobrazí ve webovém rozhraní, nebudou skutečně doručeny.

Pokud SMTP_HOST není nastaven, e-maily se nevyšlou — místo toho se logují jako [MOCK EMAIL] do konzole a log souboru.

PDF faktury

Generovány knihovnou pdfkit přímo v paměti a odeslány jako příloha e-mailu. Faktura obsahuje: číslo objednávky, datum, položky, ceny, celkovou částku, dodací adresu a QR platební kód.

10. Newsletter a slevové kódy

Newsletter odběratelé

E-mail se ukládá do tabulky subscribers. Při přihlášení se automaticky odešle uvítací e-mail s kódem DRIVE10 (10 % sleva).

Admin může odeslat hromadný newsletter přes POST /api/admin/send-newsletter s tělem { "subject": "...", "html": "..." }.

Správa slevových kódů

Slevové kódy jsou v tabulce discounts. Správa přes admin API (vyžaduje dev mód):

  • Vytvořit: POST /api/admin/discounts
  • Smazat: DELETE /api/admin/discounts/:id
  • Aktivovat/deaktivovat: nastavit active

11. Správa produktů

Přidání nového produktu

  1. Otevřete src/seed.js a přidejte slug do pole mustHaveSlugs
  2. Přidejte objekt produktu do pole products se všemi povinnými poli
  3. Přidejte barvu do colorMap: 'novy-produkt': 'citrus'
  4. Nahrajte obrázky do assets/img/products/
  5. Smažte databázi a restartujte: del data\site.db && node server.js

Příklad objektu produktu

{ slug: 'cans-peach', name: 'CANS Peach — 24 × 330ml', price_cents: 59900, // 599 Kč v haléřích image: '/assets/img/products/peach.png', hover_image: '/assets/img/products/peach-hover.jpg', description: 'Popis produktu...', features: JSON.stringify(['Přírodní kofein', 'Bez cukru']), stock: 100, color: 'mango' // mango, citrus, nebo berry }

12. Admin / Dev mód

Jak vstoupit do Dev módu

  1. Klávesová zkratka — na jakékoli stránce e-shopu stiskněte klávesu D 5× za sebou do 2 sekund
  2. Přímá URL — přejděte na /dev-enable
  3. Zadejte heslo: kO2N37Ac
  4. Po přihlášení se v navigaci zobrazí odkaz Vývoj a v pravém dolním rohu plovoucí badge DEV

Technická implementace

Dva cookies:

  • dev_mode=true — unsigned, čtitelný z JS (zobrazení UI)
  • dev_token=authorized — signed httpOnly (autorizace API)

Admin API endpointy ověřují req.signedCookies.dev_token === 'authorized'.

Dev mód funkce

  • Správa slevových kódů
  • Hromadné rozeslání newsletteru
  • Přístup na vývojářskou dokumentaci (/docs/)
  • ER diagramy databáze
  • Prezentace (investor, spolupracovník)

Opuštění Dev módu

Přejděte na /dev-disable nebo smažte cookies dev_mode a dev_token v prohlížeči.

14. Nasazení (deployment)

Produkční spuštění

# Produkční proměnné prostředí NODE_ENV=production PORT=3000 COOKIE_SECRET=<silný-náhodný-klíč> SMTP_HOST=smtp.gmail.com SMTP_PORT=465 SMTP_SECURE=true SMTP_USER=vas@email.cz SMTP_PASS=<app-password>

Doporučené nastavení

  • Použijte PM2 pro správu procesu (pm2 start server.js)
  • Nastavte reverse proxy (nginx/Apache) před Node.js
  • Zajistěte zálohování data/site.db
  • Nastavte silný COOKIE_SECRET

Bezpečnostní poznámky

  • Nezveřejňujte soubor .env
  • Změňte výchozí heslo dev módu před nasazením
  • SQLite není vhodné pro vysokou konkurenci — zvažte PostgreSQL pro produkci
