// 01 — O projektu

CO JE DASH! DIE! REPEAT!

Popis hry

Blood Dash je top-down akcni hra ve fullscreen rozliseni 1920x1080 px vytvorena v Pygame. Hrac ovlada postavu schopnou provadet utocny dash (uskok) smerem k kurzoru mysi. Cilem je prezit co nejdele vlnu neustale se mnozicich nepriatel a dosahnout co nejvyssiho skore.

Herni mechaniky

Pohyb hrace pomoci WASD, utok levym tlacitkem mysi (Dash). Kazdy dash zanechava krvavou stopu a nici vsechny nepratele, ktere zasahne. System combo retezeni a Frenzy Mode — specialni stav zrychlujici hrace a zpomalujici nepratele.

Webova prezentace

Projekt doplnuje plnohodnotna webova aplikace v Django frameworku. Obsahuje registraci a prihlaseni uzivatelu, profil s ulozene high-score, interaktivni pruvodce hrou, ER diagram a dokumentaci projektu. Skore se po kazde hre automaticky odeslani na server pres REST API.

Technologie

Python 3 Pygame 2.6 Django 5.2 SQLite REST API HTML/CSS Threading Git
// 02 — Klicove funkce

CO UMIME

01

Prihlaseni uzivatele

Registrace, prihlaseni a odhlaseni pres Django Auth. Ruzny obsah pro prihlaseneho/neprihlaseneho uzivatele — profilova stranka s high-score jen pro prihlasene.

02

Databazova data na webu

Profilova stranka zobrazuje ulozene high-score z databaze. Admin rozhrani Django umoznuje mazat zaznamy primo z prohlizece. Email uzivatele lze aktualizovat pres formular.

03

REST API & Threading

Po kazde hre Pygame odeslani skore na /api/update_score/ pomoci samostatneho vlakna — hra se nezasekne. Server ulozi skore pouze pokud je vyssi nez dosavadni rekord.

04

Graficke rozhrani Pygame

Fullscreen 1920x1080, sprite animace (5 smeru pohybu + idle), particle system krevnich stop s fade efektem, frenzy overlay s pulsujicim efektem.

05

Adaptivni obtiznost

Rychlost nepriatel se stupnuje pri 50 a 100 bodech. Frenzy Mode se aktivuje po 3 kombo retezecich — zmeni rychlosti hrace i nepriatel a zkrati cooldown dashe.

06

Dokumentovany kod

Tridy a funkce jsou okomentovany. Pygame i Django casti dodrzuji konvence Pythonu. Modularni struktura: main.py, menu.py, views.py, models.py, urls.py.

JAK TO FUNGUJE

PSEUDOKOD — DASH & COMBAT SYSTEM

PROCEDURE HandleDash(player, enemies, mouse_pos, current_time) IF LMB_pressed AND (current_time - last_dash_time) >= dash_cooldown THEN direction ← normalize(mouse_pos - player.pos) start_pos ← player.pos // Krevni stopa — 8 blood splatu rovnomerne podel trasy FOR i IN range(0, 8) DO SpawnBloodSplat(start_pos + direction × i × 35) END FOR player.pos ← player.pos + direction × dash_distance last_dash_time ← current_time dash_rect ← BoundingBox(start_pos, player.pos, padding=64) combo ← 0 FOR EACH enemy IN enemies DO IF Collides(dash_rect, enemy.rect) THEN RemoveEnemy(enemy); combo ← combo + 1 END IF END FOR score ← score + min(combo, 5) IF combo >= 5 THEN combo_count ← combo_count + 1 IF combo_count >= 3 THEN ActivateFrenzy() END IF END IF END IF END PROCEDURE

PSEUDOKOD — POHYB NEPRIATEL (HOMING AI)

PROCEDURE UpdateEnemies(enemies, player.pos, enemy_speed) FOR EACH enemy IN enemies DO delta ← player.pos - enemy.pos distance ← hypot(delta.x, delta.y) IF distance > 0 THEN direction ← delta / distance // normalizovany vektor enemy.pos ← enemy.pos + direction × enemy_speed END IF IF Collides(player.rect, enemy.rect) THEN SendScoreAsync(username, score) // threading RETURN DeathScreen(score) END IF END FOR END PROCEDURE

PSEUDOKOD — UPDATE SCORE API (DJANGO)

FUNCTION update_score(POST request) data ← parse_json(request.body) username ← data["username"]; new_score ← data["score"] user ← User.get(username=username) // raises DoesNotExist → 404 profile ← Profile.get_or_create(user=user) IF new_score > profile.score THEN profile.score ← new_score profile.save() RETURN JSON {status: "success", new_highscore: true} ELSE RETURN JSON {status: "success", new_highscore: false} END IF END FUNCTION

CREATE TABLE — SCHEMA

SQL CREATE TABLE myapp_profile ( id INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT, user_id INTEGER NOT NULL UNIQUE REFERENCES auth_user(id) ON DELETE CASCADE, score INTEGER NOT NULL DEFAULT 0, email VARCHAR(254) NOT NULL DEFAULT '' );

SELECT — ZEBRICEK HRACU (JOIN + ORDER + FILTER)

SQL -- Zebricek: top 10 aktivnich hracu se skore > 0 SELECT u.username, u.email, p.score, u.date_joined FROM myapp_profile p INNER JOIN auth_user u ON p.user_id = u.id WHERE p.score > 0 AND u.is_active = 1 ORDER BY p.score DESC LIMIT 10; -- Skupinova statistika (GROUP BY) SELECT COUNT(u.id) AS total_players, AVG(p.score) AS avg_score, MAX(p.score) AS top_score FROM myapp_profile p JOIN auth_user u ON p.user_id = u.id WHERE u.is_active = 1; -- UPDATE — aktualizace high-score UPDATE myapp_profile SET score = 217 WHERE user_id = (SELECT id FROM auth_user WHERE username = 'BloodRun') AND 217 > score; -- DELETE CASCADE: smazani uzivatele automaticky smaze i profil DELETE FROM auth_user WHERE username = 'DashQueen';
