Přehled pro nového spolupracovníka

Rychlá orientace v projektu: co je kde, jak spolu části mluví a kde začít, když budeš něco upravovat.

1. Struktura projektu

  • Frontend je tvořený statickými HTML a CSS soubory v kořeni projektu.
  • Backend běží ve složce backend a používá Flask + SQLite.
  • Herní logika je v snake_game.py, assets jsou v images.
  • Společné styly a navigace jsou sdílené mezi stránkami přes O hře.css.

2. Backend

  • Flask aplikace v backend/app.py.
  • Databáze je uložená v backend/db.sqlite3.
  • API řeší registraci, login, profil a ukládání skóre.
  • Token se ukládá do databáze a používá se jako jednoduché přihlášení.

3. Frontend

  • Stránky jako Přihlášení, Registrace, Můj účet, TOP10 jsou čistě HTML/CSS.
  • Navigace je stejná napříč webem, takže změny se obvykle kopírují do více stránek.
  • Dropdown Dokumentace vede na manuál, investorskou prezentaci a tento přehled.
  • Prohlížečová část je navržená jako lehký web bez frameworku.

4. Tok dat

  • Uživatel se zaregistruje nebo přihlásí přes API.
  • Frontend uloží token do localStorage.
  • Stránka Můj účet čte profil a nejlepší skóre.
  • Hra posílá skóre zpátky do backendu přes endpoint pro výsledky.

5. Kde začít

  • Pokud měníš UI, začni v HTML stránkách a následně dolaď CSS.
  • Pokud měníš přihlášení nebo skóre, koukni do backend/app.py.
  • Pokud měníš grafiku nebo spritey, uprav images a assets.py.
  • Pokud se ti něco rozbije, nejdřív zkontroluj cesty k souborům a názvy endpointů.

6. Rychlé spuštění

  • Backend: python backend/app.py
  • Frontend: otevřít přes Live Server z kořene projektu
  • Hra: python snake_game.py
  • Obrázky jsou v jedné společné složce images
